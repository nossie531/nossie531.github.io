'_' (型ホルダー) の未対応箇所について。

関連エラー

以下のエラーはこの仕様に関連している。

  • [error] '_' の const や static での使用
  • [E0121] '_' の未対応箇所での使用

基本ルール

アイテム (パスを持つ構成要素) の型の記述では '_' を使った型推論はできない。

特に以下はうっかりしやすい。

  • 関数の返値型
  • const と static

個人的な感想。

妥当な仕様ではあるのだけど、配列の要素数を手動で指定する必要がある点だけは、何とかしてほしい。一応、const や static でも '_' による型推論を可能にしようという RFC は提案されているが、いますぐ採用されるような流れではない。
