Oops! The debug adapter has terminated abnormally.

VSCode でデバッグ中に CodeLLDB が落ちると上記のエラーメッセージが出る。このエラーメッセージ自体には様々な原因が考えられるが、ここでは 2026 年現在の Windows 環境で最も多いであろう原因についてメモしておく。

背景にある相性問題

Windows 環境では定番の Rust ツールチェーンの MSVC は CodeLLDB と相性があまりよくないらしい。そもそも、LLDB は MSVC に非対応だが、CodeLLDB はそれを独自にバンドルして動かしているのだとか。

こうした背景のため、ごく簡単な操作でもエラーになる事があるようだ。とは言え、この解決のためだけに MinGW を導入して GNU ツールチェーンを使ったり、WSL 等の Linux 環境を用意するのは少し面倒だろう。

再現手順

未初期化の変数にマウスホバーするだけで落ちてしまう。そのため例えば、以下のコードでは最初の let 文の実行前にデバッガで停止し、result にマウスホバーするだけで落ちる。 


fn main() {
    let result = func();
    println!("{result}");
}

fn func() -> usize {
    42
}

解決策

VS Code の設定ファイル setting.json に以下の行を記述をするといい。これでホバー時のポップアップを無効化できる。使いづらくはなるが、デバッガが落ちるよりはましだろう。 


"lldb.evaluateForHovers": false,
