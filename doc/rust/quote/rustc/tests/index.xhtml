--nocapture

stdout と stderr をキャプチャせず、テストにコンソールへのプリントを許可する。通常は出力はキャプチャされ、テストが失敗した場合のみ表示される。

これは RUST_TEST_NOCAPTURE 環境変数を 0 以外に設定することでも指定できるだろう。

--show-output

stdout と stderr を全てのテストの実行後に表示する。

--nocapture はテストへそれらの実行中のプリントを許可しており、これはもし複数のテストが存在し並列に実行されていると出力が交互配置されうるが、対照的に --show-output は出力が連続するのを保証する、ただし全てのテストの終了の待機が必要。
